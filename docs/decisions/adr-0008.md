# ADR-0008 — Tool, profile, and spec versioning

## Decision
The invariant is that every git commit must be consistent: the repo validates cleanly per commit, with profile and tool versions pinned in-repo so they agree within each commit. Refined by D-008's successor decision D-012: for brownfield, "consistent" means clean relative to a committed baseline. See requirement VL-012.

## Context
Promoted from decision D-008 in the IBuildOS master requirements specification (2026-06-30).
