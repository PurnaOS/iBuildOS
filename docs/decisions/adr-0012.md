# ADR-0012 — Brownfield baseline and ratchet

## Decision
Existing repos adopt incrementally. A committed baseline records accepted pre-existing debt; the gate blocks only new or changed artifacts and the baseline only shrinks (ratchet). "Consistent" (VL-012 and D-008) means clean relative to the baseline. See requirements VL-013 and VL-014.

## Context
Promoted from decision D-012 in the IBuildOS master requirements specification (2026-06-30).
