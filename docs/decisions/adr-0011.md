# ADR-0011 — Roles and authorization

## Decision
Deferred and out of scope for now. Identity is handled lightly via User and Team entities (WP-009) for assignment and notification only, with no permissions, role-based access, or approval-authority model. This may be revisited.

## Context
Promoted from decision D-011 in the IBuildOS master requirements specification (2026-06-30).
