IBuildOS shall be able to **trigger and run tests** — orchestrating the project's automated test runners and guiding manual test execution — then capture the results and store them as OKF artifacts (`TT-008`). It is runner-agnostic and orchestrates existing tools (cf. `CQ-001`) rather than embedding a test framework. (also runnable from the UI, `UI-013`)
