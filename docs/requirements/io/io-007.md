title: "Incumbent coexistence during migration"
owner: srini
status: draft

priority: must
---

Beyond one-shot import (`IO-004`), the system shall support **incremental re-import and a read-only mirror** of incumbent systems (Jira/Confluence/etc.) during transition — a stable external-ID link on each imported artifact, idempotent re-sync of changed source items, and optional outward status reflection — so a team can run IBuildOS alongside incumbents for months without dual data entry or a hard cutover. Two-way write-back stays opt-in; read-mirror is the default.
