The repository should validate cleanly at **every commit** — meaning clean **relative to a committed baseline** of accepted pre-existing debt (`VL-013`): no *new* violations are introduced even if legacy debt remains. On a greenfield repo the baseline is empty, so this is full consistency. The active profile and tool versions shall be pinned in-repo so tool, profile, and OKF-spec versions stay consistent within a commit. (decisions D-008, D-012)
