The linter shall support a committed **baseline** recording accepted pre-existing violations (by rule + artifact + fingerprint), reported as informational debt but **not** failing the gate, plus a **changed-artifacts-only** mode so the per-commit gate (`VL-012`) blocks only *new or modified* artifacts. The baseline may only shrink (ratchet); a command regenerates/refreshes it. (decision D-012)
