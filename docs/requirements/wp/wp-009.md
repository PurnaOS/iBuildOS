Define **`User`** and **`Team`** as first-class artifact types — a `User` carries a name and git user identity (optionally email/handle); a `Team` groups Users. Work items are **assigned** to a User or Team via an `assignee` relationship, and ownership/assignment reference these entities (`WP-007`, `WP-008`). This is lightweight identity for assignment and notification — **not** a permissions, role, or approval model (those are deferred; see decision D-011).
